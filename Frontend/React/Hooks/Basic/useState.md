## useState란?
- 함수형 컴포넌트에서 상태를 관리하기 위한 훅
---
## 사용해야 하는 이유
- 함수형 컴포넌트는 렌더링될 때마다 함수가 재실행 되는데, 이를 변수만으론 상태 유지가 힘듦.
---
## 주요 특징
- **함수 컴포넌트에서 상태 관리**
>클래스 컴포넌트에서만 가능했던 상태 관리를 함수 컴포넌트에서도 가능하게 해준다.

- **자동 UI 업데이트**
>상태가 변경되면 React가 자동으로 해당 컴포넌트를 다시 렌더링하여 UI를 최신 상태로 유지해준다.

- **초기값 설정**
>useState를 호출할 때 초기 상태 값을 설정해줘야 한다.

## 기본 사용법
```tsx
import { useState } from "react";

const Component1 = () => {
  const [value, setValue] = useState(0); // ()안에는 0, "", null 등이 올 수 있다.
};
```

## 사용 시 주의점

### 상태는 즉시 변경되지 않는다
- setState는 즉시 값을 바꾸지 않고 스케줄링된다.
- 한 렌더링 동안 상태 값은 고정된다.
### 이전 상태 기반 변경은 updater 함수 사용
```tsx
setValue((prev) => prev + 1);
```
## 기억 포인트
- 상태는 렌더링 단위로 고정된다
- setState는 비동기처럼 동작한다
- 이전 값이 필요하면 updater 함수를 쓴다
