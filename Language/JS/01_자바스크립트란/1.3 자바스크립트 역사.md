>초창기 자바스크립트는 웹페이지의 보조적인 기능을 수행하기 위한 용도로 사용됨.

## 1999
- 서버와 브라우저가 `비동기(Asynchronous)` 방식으로 데이터를 교환할 수 있는 `Ajax(Asynchronous JavaScript and XML)`가 XMLHttpRequest라는 이름으로 등장함.
- Ajax를 통해 서버로부터 필요한 **데이터**만 전송받아 변경해야 하는 부분만 한정적으로 `렌더링`하는 방식이 가능해짐.
- `데스크톱 애플리케이션`과 **유사한 빠른 성능과 부드러운 화면 전환**이 가능해짐.
## 2005
- 구글이 `구글 맵스`를 발표하면서, 자바스크립트는 **웹 애플리케이션 프로그래밍 언어**로서 가능성이 있다는 것을 보여줌.
- 구글 맵스 : **자바스크립트와 Ajax**를 기반으로 동작하여 **빠른 성능과 부드러운 화면 전환 효과**를 보여줌.
## 2006
- **DOM(Document Object Model)**를 쉽게 제어할 수 있는 `jQuery`가 등장함.
- 배우기 까다로운 자바스크립트보다 **배우기 쉽고 직관적인** jQuery를 선호하는 개발자가 많아짐.
## 2008
- 구글이 더욱 빠르게 동작하는 `V8 자바스크립트 엔진`을 개발함.
- **V8 자바스크립트 엔진**을 통해 웹 서버에서 수행되던 로직들이 대거 **클라이언트(브라우저)**로 이동함.
## 2009
- **라이언 달(Ryan Dahi)**이 브라우저 이외의 환경에서도 동작이 가능한 `Node.js`를 발표함.
- 주로 서버 사이드 애플리케이션 개발에 사용됨.
## 그외
- 빠른 성능과 사용자 경험을 제공하는 것이 필수가 되며, 개발 규모와 복잡도가 상승하여
  `CBD(Component based development)` 방법론을 기반으로 하는 `SPA(Single Page Application)`가 대중화됨.
- 대표적인 프레임워크 라이브러리 : React, Vue.js, Angular, Svelte 등
