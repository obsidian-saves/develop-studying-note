## 1.리액트에 대해 설명해주세요.
>리액트는 자바스크립트 **라이브러리**의 하나로서 사용자 인터페이스 (UI)를 만들기 위해 사용됩니다.

|  구분   |     `라이브러리`     |      `프레임워크`      |
| :---: | :-------------: | :---------------: |
|  역할   |    특정 기능 제공     |  애플리케이션 전체 구조 제공  |
|  제어권  |       개발자       |       프레임워크       |
| 구조 강제 |     X (없음)      |      O (있음)       |
| 사용 방식 |    필요한 것만 선택    |   정해진 규칙에 따라야함    |
|  종류   | React, jQuery.. | Angular, Spring.. |
### 1-1. 리액트를 사용하는 이유에 대해 설명해주세요.
>리액트를 사용하는 이유는 크게 다음과 같습니다.

- **컴포넌트 기반의 화면 구성**
	>리액트는 화면의 한 부분을 컴포넌트 라는 **단위**로 나눌 수 있으며 **독립적으로 관리**할 수 있습니다.
	>대규모 **웹 애플리케이션**에서 컴포넌트는 역할과 기능에 따라 따로 관리하기 용이하며,
	>반복되는 부분을 대체할 수 있게 해주어서 **코드 재사용성**을 높여줍니다.
	>또한 컴포넌트 기반의 화면을 구성한다면 블록 쌓기처럼 컴포넌트를 쌓아서 **빠르고 효율적으로** 화면을 구현할수 있습니다.

-  **[[Virtual DOM]]을 통한 효율적인 렌더링**
	>**Virtual DOM**은 UI의 이상적인 또는 "가상"적인 표현을 **메모리에 저장**하고,
	>**React DOM**과 같은 라이브러리에 의해 "실제" DOM과 동기화하는 **프로그래밍 개념**입니다.
	>이 과정을 **재조정**이라고 합니다.
	>
	>이 접근방식은 React의 **선언적 API**를 가능하게 합니다.
	>React에게 원하는 UI의 상태를 알려주면 DOM이 그 상태와 일치하도록 합니다.
	>이러한 방식은 앱 구축에 사용해야 하는 어트리뷰트 조작, 이벤트 처리, 수동 DOM 업데이트를 추상화합니다.  

- **풍부한 생태계**
	>리액트는 **2013년** 출시 이후 꾸준한 인기를 누려왔으며, 특히 2010년대 중후반부터 현재까지 프론트엔드 개발 표준으로 자리잡게 되었습니다.
	>특히 **2016년도** **Angular**의 인기가 떨어지게 되면서,
	>리액트는 프론트엔드 개발자들에게 빠르게 확산되어 여러 라이브러리 등이 대거 등장하게 되었습니다.
