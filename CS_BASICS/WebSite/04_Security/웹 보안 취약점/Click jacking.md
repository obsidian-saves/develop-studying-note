## 의미
> 사용자가 인지하는 것과 다른 동작을 하도록 사용자 인터페이스(UI)를 속이는 웹 보안 공격을 의미한다.

예를 들어, 페이스북에 마케터로 위장한 해커가 게시물을 올렸다고 가정하여 FaceBook 페이지에서 좋아요를 얻기 위한 게시물을 만든다. 하지만 좋아요 버튼을 클릭하면 멀웨어의 무단 다운로드, JavaScript 코드를 트리거하여 웹캠을 켜거나 암호 수집 등이 가능해진다.

사용자가 보고 있는 웹사이트는 누구에게나 신뢰받고 안전하다고 느껴지는 FaceBook같은 실제 기업의 웹사이트다. 하지만 해커는 맞춤형 캐스케이팅 스타일 시트(CSS)를 통해 iframe을 포함한 다양한 HTML 기술을 사용하여 콘텐츠에 보이지 않는 오버레이를 추가하며, 이를 통해 다른 웹사이트의 콘텐츠를 다른 웹사이트로 옮길 수 있다.

또다른 버전인 _Cursor jacking_ 도 있다.

_Cursor jacking란_
사용자가 클릭할 의도가 없는 페이지의 일부를 클릭하도록 혼동하는 사용자 지정 커서 이미지를 추가하여 사용자를 속이는 방식이다. 사용자 이름, 암호, 신용카드 번호 및 기타 개인 정보를 자주 사용하는 일반적인 사이트에 입력시킬 수도 있다.

## 공격 유형

**완전한 투명 오버레이**
일반적으로 합법적인 웹 페이지를 악성 페이지에 오버레이하여, 합법적인 페이지가 보이지 않는 iframe에 로드되고 사용자는 악성 페이지가 아래에 있는지 알지 못하게 하는 방식이다.

**자르기**
해커가 악성 페이지에서 합법적인 페이지로 선택한 제어만 오버레이할 때 발생시켜 합법적인 페이지의 하이퍼링크를 리디렉션으로 대체하거나, 버튼 텍스트를 다른 언어로 대체하여 사용자를 오도하는 방식으로 콘텐츠를 변경시킬 수 있는 방식이다.

**숨겨진 오버레이**
앞서 이야기한 _Cursor jacking_ 이 여기에 속한다. 해커는 1x1 픽셀 정도의 작은 iframe을 생성하여 마우스 커서 아래에 위치를 두어 사용자가 탐지할 수 없게 하는 방식이다.

**이벤트 드롭 클릭**
해커가 CSS 포인터 이벤트  속성을 없음으로 설정하여 페이지에 아무것도 없는 것처럼 보이게 하지만 실제론, 클릭이 아래의 악성 페이지에서 작동하도록 하는 방식이다.
웹 사이트의 버튼이나 링크를 계속 클릭하는 것이 작동하지 않으면 사이트 담당자 혹은 보안팀에게 연락을 취해야한다.

**신속한 콘텐츠 교체**
구성표에서 오버레이가 가려지고 1초 동안 제거되어 클릭을 다른 악성 버튼이나 링크로 즉시 교체하는 방식이다. 사용자 경험과 행동에 대한 상당한 노하우를 가지고 있는 해커가 사용할 수도 있다.

**스크롤**
합법적인 대화 상자 또는 화면 일부에서 버튼이 있는 팝업을 생성해, 버튼 아래의 무해해 보이는 프롬프트 안에 악성 웹 페이지로 이동하게 하는 방식이다.
허나 사용자가 브라우저에 광고 차단 확장 프로그램를 설치하였을 수도 있어, 해커는 이를 우회할 방법을 찾아한다. (Bogus 광고 차단 기능 확장은 또 다른 공격이다.)

**재배치**
사용자가 웹 페이지의 다른 부분에 집중하는 동안 해커가 신뢰할 수 있는 사용자 인터페이스(UI) 요소를 빠르게 만들어 사용자가 이 요소를 클릭하게 만드는 방식이다.
대부분 사용자에게는 빠른 점프나 움직임을 캐치할 수 없어, 이런 일이 발생한다면 사이트 담당자 혹은 보안팀에게 알려야한다.

**드래그 앤 드롭 기능**
해커는 사용자가 눈치채지 못하도록 합법적으로 보이는 웹 양식 화면을 만들어 사용자에게 필드를 작성하게 하거나, 요소를 드래그·이동하도록 유도하는 방식이다. 이 과정에서 입력된 정보나 드래그된 데이터는 실제로 아래에 숨겨진 악성 페이지를 통해 탈취될 수 있다.
