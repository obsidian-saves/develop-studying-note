개별적인 코드 단위(함수, 메서드)가 의도한 대로 작동하는지 확인하는 과정.
개발 코드 단위를 테스트하여 오류 발견 -> 수정 -> 소프트웨어 품질 향상시키는 과정.

---
## 예시
단위 테스트 작성을 위해서는 다양한 라이브러리와 프레임워크를 제공한다. 예를 들어 Java에서는 Junit이 있다.

```java
public class Calculator {
	public int add(int a, int b) {
	return a + b;
	}
}

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {
@Test
public void testAdd() {
		Calculator calculator = new Calculator();
		int result = calculator.add(2, 3);
		assertEquals(5, result);
	}
}
```
위 코드에서 `Calculator` 클래스는 두 개의 정수를 더한 결과 값을 반환한다. `CalculatorTest` 메서드에서 Junit의 `@Test` 어노테이션을 사용하여 `testAdd` 메서드를 선언한다. `testAdd` 메서드를 통해 Calculator의 `add` 메서드가 제대로 작동하는지 확인할 수 있다.

`assertEquals` 메서드를 통해 `testAdd` 메서드에서 Calculator 클래스의 `add`의 메서드 반환 값을 비교하여 만약 값이 같다면 테스트 성공, 다르면 실패한다.
