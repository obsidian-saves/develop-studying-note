>소프트웨어의 기능과 동작을 테스트하는 데 사용하는 코드로, 개발자가 코드를 실행하였을 때, 예상한 결과값이 제대로 나오는 지 확인하는 데 쓰인다.

각 단계의 테스트 코드는 [[V Model ❌]]을 통해 거쳐간다.

## 종류

* **단위 테스트([[1. Unit Testing]])**

* **통합 테스트([[2. Intergration Testing ❌]])**

* **시스템 테스트([[3. System Testing ❌]])**

* **사용자 인수 테스트([[4. User Acceptance Testing ❌]])**

각각의 테스트는 특정한 측면에서 소프트웨어를 평가한다.

모든 테스트는 기대한 입력값과 출력값을 반환하는지 확인해야한다.
