>프로젝트가 빌드되고 실행하기 위해 필요한 외부 라이브러리를 뜻한다.

예를 들어 웹 개발을 하기 위해서는 spring-boot-start-web이라는 종속성이 필요하다.

종속성의 관리는 관리 도구인 `Gradle`과 `Maven`이 라이브러리를 다운로드, 관리한다.

### 이점
* Spring Boot 버전을 한 곳에서 지정하여 종속성 정보의 중앙 집중화를 제공한다.
* 한 버전에서 다른 버전으로 전환할 때 도움된다.
* 여러 버전의 스프링 부트 라이브러리가 일치하지 않는 것을 방지한다.
* [[다중 모듈 ❌]] 프로젝트에 유용하다.

### 동작 원리
1. Gradle 또는 Maven이 [[repositories ❌]]에서 의존성 다운로드
2. Starter가 포함하는 여러 라이브러리까지 자동으로 다운로드 (트랜스티브 종속성)
3. 프로젝트 빌드시 classpath에 포함되어 사용 가능

## Maven의 종속성 관리
Maven은 [[선언형 빌드 시스템 ❌]]으로서, [[pom.xml ❌]]이라는 XML 파일에 모든 설정을 명시한다.
표준화, 명확성이 크게 강조되어, 기업 시스템에서 오랫동안 많이 사용되었다.

### 선언 예시
pom.xml 안에 `<dependencites>`를 추가한다.
```xml
<dependencies>
  <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <version>3.2.0</version>
  </dependency>
</dependencies>
```
위 코드와 같이 선언형 구조라 누구나 XML을 보면 상태 파악이 쉬워진다. 의존성 버전이 충돌해도 상대적으로 관리가 보수적이다.

## Gradle의 종속성 관리
Gradle은 스크립트 기반 빌드 시스템으로서, [[build.gradle]] ( [[Groovy ❌]] ) 또는 build.gradle.kts ( Kotlin DSL ) 파일에 모든 설정을 명시한다.

### 선언 예시
```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web:3.2.0'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}
```
`dependencies {}` 블록 안에 모든 의존성을 나열한다.

선언이 간결하고 코드 가독성이 높다.

코드 기반의 파일이므로 반복문, 조건문이 사용 가능하다.

예를 들어 아래와 같이 여러 의존성을 반복문으로 묶을 수 있다.
```groovy
def commonLibs = [
    'org.projectlombok:lombok:1.18.30',
    'org.springframework.boot:spring-boot-starter-validation'
]

dependencies {
    commonLibs.each { implementation it }
}
```
